秒杀接口优化思路：
1. 系统初始化，将商品库存数量加载到Redis
2. 收到库存，Redis预减库存，库存不足，直接返回，否则进入3
3. 请求进入消息队列中，返回排队中（异步处理）
4. 请求出队，生成订单，减少库存
5. 客户端轮询，是否秒杀成功

这块代码需要再修改，减库存并发部分要保证线程安全
